## Development Guidelines

git commitするとき以下に注意すること

* 変更内容が混在している場合は、git add -pで必要な部分のみをステージングし、テーマごとにcommitを分けること
* ステージング後にgit diff --cachedで確認し、本来の作業と無関係な変更が含まれていたら、commit前に依頼者に確認すること
* commitメッセージは1行しか表示されないため、説明は1行に収めること（Co-Authored-By: など追加要素は2行目以降で良い）
* commitメッセージは, リポジトリが `karia/` 配下であれば、英語で記述すること。それ以外のリポジトリの場合は過去のコミットメッセージと同様の言語で記述すること。
* commitはなるべく細かい単位で実施すること（細かすぎる場合はあとでまとめるので、気にせず細かくしてよい）
* commit時にpre-commitの警告をskipする場合、skipしてよいかを理由込みで依頼者に確認すること（確認が必要だからpre-commitを設定しているので、通常skipすべきではない）

IssueやPull Requestを作成するとき以下に注意すること

* descriptionの記述量をミニマムに保つこと。多すぎると読んでくれません。特にコードを読めばわかることは記述しない。
* テンプレートが用意されている場合はテンプレート通りに記入すること。
  * Pull Requestのテンプレートファイル名は通常ルートディレクトリの `pull_request_template.md` であるが、サブディレクトリ配下にあるケースもある
  * テンプレートの項目は自己判断で削除してはならない。記入不要な項目は空欄のまま残すこと。ただし、テンプレートに削除するよう指示がある場合は例外とする。
* title,descriptionを記述するときは以下の言語を使用する。
  * リポジトリが `karia/` 配下の場合は全て英語表記で統一する
  * 上記以外のリポジトリ、特にOrganization配下のリポジトリの場合、日本人がレビュアーであるため特別な指示がなければ日本語で記述すること
* 作成を完了したら、内容を確認するためopenコマンドを使ってURLを開くこと
